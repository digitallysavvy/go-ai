# xAI Provider

xAI provides Grok models with real-time knowledge through X (Twitter) integration, image/video generation, and strong reasoning capabilities.

## Setup

### Installation

```go
import (
    "github.com/digitallysavvy/go-ai/pkg/ai"
    "github.com/digitallysavvy/go-ai/pkg/providers/xai"
)
```

### Configuration

```go
provider := xai.New(xai.Config{
    APIKey: os.Getenv("XAI_API_KEY"),
})

// Default: Responses API (recommended)
model, err := provider.LanguageModel("grok-3")

// Opt-in: Chat Completions API (legacy)
chatModel, err := provider.ChatCompletionsLanguageModel("grok-3")
```

### Get API key

1. Sign up at [x.ai](https://x.ai)
2. Request API access
3. Set environment variable:

```bash
export XAI_API_KEY=xai-...
```

## Responses API (default)

Starting in v0.4.0, `LanguageModel()` returns a Responses API model. This is the recommended API for new development. Use `ChatCompletionsLanguageModel()` if you need the legacy Chat Completions API.

```go
model, _ := provider.LanguageModel("grok-3")

result, _ := ai.GenerateText(ctx, ai.GenerateTextOptions{
    Model:  model,
    Prompt: "What's happening in AI today?",
})
```

## Provider options

### Reasoning summary

Control reasoning output in Responses API:

```go
result, _ := ai.GenerateText(ctx, ai.GenerateTextOptions{
    Model:  model,
    Prompt: "Explain quantum computing",
    ProviderOptions: map[string]interface{}{
        "xai": map[string]interface{}{
            "reasoningSummary": "detailed", // "auto", "concise", or "detailed"
        },
    },
})
```

### Logprobs

```go
result, _ := ai.GenerateText(ctx, ai.GenerateTextOptions{
    Model:  model,
    Prompt: "Hello",
    ProviderOptions: map[string]interface{}{
        "xai": map[string]interface{}{
            "logprobs":    true,
            "topLogprobs": 5,
        },
    },
})
```

## Image generation

```go
imageModel, _ := provider.ImageModel("grok-2-image")

result, _ := ai.GenerateImage(ctx, ai.GenerateImageOptions{
    Model:  imageModel,
    Prompt: "A futuristic cityscape at sunset",
    ProviderOptions: map[string]interface{}{
        "xai": map[string]interface{}{
            "quality": "high",  // "low", "medium", "high"
        },
    },
})
```

Supports multiple input images for editing, b64_json output format, and `CostInUsdTicks` in provider metadata.

## Video generation

```go
videoModel, _ := provider.VideoModel("grok-video")

result, _ := ai.GenerateVideo(ctx, ai.GenerateVideoOptions{
    Model:  videoModel,
    Prompt: "A cat playing piano",
})
```

Returns `ModerationError` if content is filtered. `CostInUsdTicks` available in provider metadata.

## Best practices

1. **Use the Responses API** — it is the default and recommended path
2. **Leverage ReasoningSummary** — get insight into model reasoning without full chain-of-thought overhead
3. **Check CostInUsdTicks** — available in image and video provider metadata for cost tracking
